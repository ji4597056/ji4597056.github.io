### 动态规划（DP）

&emsp;&emsp;**什么是动态规划？**动态规划的本质是对问题状态的定义和状态转移方程的定义。动态规划通过拆分问题，定义问题状态和状态之间的关系，使得问题能够以**递推**（或者说分治）的方式去解决。

&emsp;&emsp;从上述描述可以看出，**拆分问题**才是动态规划的核心。而拆分问题，靠的就是**状态的定义**和**状态转移方程的定义**。

### leetcode 198（House Robber）

&emsp;&emsp;`leetcode`第198题：你是一个专业的小偷，计划偷窃沿街的房屋。每间房内都藏有一定的现金，影响你偷窃的唯一制约因素就是相邻的房屋装有相互连通的防盗系统，如果两间相邻的房屋在同一晚上被小偷闯入，系统会自动报警。给定一个代表每个房屋存放金额的非负整数数组，计算你在不触动警报装置的情况下，能够偷窃到的最高金额。

&emsp;&emsp;示例一:

```
	输入: [1,2,3,1]
	输出: 4
	解释: 偷窃 1 号房屋 (金额 = 1) ，然后偷窃 3 号房屋(金额 = 3)。偷窃到的最高金额 = 1 + 3 = 4 。
```
&emsp;&emsp;示例二:

```
	输入: [2,7,9,3,1]
	输出: 12
	解释: 偷窃 1 号房屋 (金额 = 2)，偷窃 3 号房屋(金额 = 9)，接着偷窃 5 号房屋(金额 = 1)。偷窃到的最高金额 = 2 + 9 + 1 = 12 。
```

&emsp;&emsp;问题很形象，题目本质相当于在给定数组中取出一个或多个不相邻数，使其和最大。那么现在以动态规划的思路来解决该问题，首先为了拆分问题，我们需要找到针对该问题状态的定义以及状态转移方程。假设从最后一家店铺开始抢，那么只会遇到2种情况，即：抢这家店和下下家店，或者不抢这家店。

- 状态：F(i)表示打劫到第i间房屋（0～i）时累计取得的金钱最大值
- 状态转移方程：F(i) = max(F(i-1), F(i-2) + nums[i])

&emsp;&emsp;根据上述分析，可以很自然地想到使用递归方法解决问题：

```java
	 // 暴力穷举
    public int rob1(int[] nums) {
        return search1(nums, nums.length - 1);
    }

    private int search1(int[] nums, int idx) {
        // 自顶向下
        if (nums.length == 0 || idx < 0) {
            return 0;
        }
        int s1 = search1(nums, idx - 2) + nums[idx];    // 选该idx
        int s2 = search1(nums, idx - 1);    // 不选该idx
        return Math.max(s1, s2);
    }

    @Test
    public void test1() {
        Assert.assertEquals(4, rob1(new int[]{1, 2, 3, 1}));
        Assert.assertEquals(12, rob1(new int[]{2, 7, 9, 3, 1}));
        Assert.assertEquals(2, rob1(new int[]{1, 2}));
    }
```

&emsp;&emsp;上述代码能够解决该问题，但缺点是性能特别差，因为涉及到大量的重复计算，每次计算F(i)，都需要计算F(i-1)和F(i-2)，由于没有保存中间结果，每次都需要递归计算一遍。下面进行优化：

```java
	 // 保存状态递归
    public int rob2(int[] nums) {
        int[] result = new int[nums.length];
        for (int i = 0; i < result.length; i++) {
            result[i] = -1;
        }
        return search2(nums, nums.length - 1, result);
    }

    private int search2(int[] nums, int idx, int[] result) {
        if (nums.length == 0 || idx < 0) {
            return 0;
        }
        if (result[idx] >= 0) {
            return result[idx];
        }
        int s1 = search2(nums, idx - 2, result) + nums[idx];
        int s2 = search2(nums, idx - 1, result);
        result[idx] = Math.max(s1, s2);
        return result[idx];
    }

    @Test
    public void test2() {
        Assert.assertEquals(4, rob2(new int[]{1, 2, 3, 1}));
        Assert.assertEquals(12, rob2(new int[]{2, 7, 9, 3, 1}));
        Assert.assertEquals(2, rob2(new int[]{1, 2}));
    }
```

&emsp;&emsp;上述代码通过使用数组保存中间计算结果解决了重复计算导致性能较差的问题。下面转换成自底向上的递推写法：

```java
	 // 保存状态递推
    public int rob3(int[] nums) {
        return search3(nums);
    }

    private int search3(int[] nums) {
        // 自底向上
        if (nums.length == 0) {
            return 0;
        }
        if (nums.length == 1) {
            return nums[0];
        }
        int[] result = new int[nums.length];
        result[0] = nums[0];
        result[1] = Math.max(nums[0], nums[1]);
        for (int i = 2; i < nums.length; i++) {
            result[i] = Math.max(result[i - 1], result[i - 2] + nums[i]);
        }
        return result[nums.length - 1];
    }

    @Test
    public void test3() {
        Assert.assertEquals(4, rob3(new int[]{1, 2, 3, 1}));
        Assert.assertEquals(12, rob3(new int[]{2, 7, 9, 3, 1}));
        Assert.assertEquals(2, rob3(new int[]{1, 2}));
    }
```

&emsp;&emsp;虽然上述递推写法和之前的递归写法时间复杂度是一样，但一般情况下，但是由于递归涉及到方法调用的开销，性能上是比递推较差的。那么还有优化的余地么？观察到状态转移方程：`F(i) = max(F(i-1), F(i-2) + nums[i])`，在递推过程中只需要保存前两项结果即可，下面使用滚动数组的技巧优化空间复杂度：

```java
    private int search4(int[] nums) {
        // 自底向上
        if (nums.length == 0) {
            return 0;
        }
        if (nums.length == 1) {
            return nums[0];
        }
        // 只用长度为2的数组
        int[] result = new int[2];
        result[0] = nums[0];
        result[1] = Math.max(nums[0], nums[1]);
        for (int i = 2; i < nums.length; i++) {
            result[i % 2] = Math.max(result[(i - 1) % 2], result[(i - 2) % 2] + nums[i]);
        }
        return Math.max(result[0], result[1]);
    }

    @Test
    public void test4() {
        Assert.assertEquals(4, rob4(new int[]{1, 2, 3, 1}));
        Assert.assertEquals(12, rob4(new int[]{2, 7, 9, 3, 1}));
        Assert.assertEquals(2, rob4(new int[]{1, 2}));
    }
```

&emsp;&emsp;上述代码已经是时间复杂度和空间复杂度权衡下的最优解，继续简化代码得到最终代码：

```java
    public int rob(int[] nums) {
        int prev = 0, curr = 0;
        for (int n : nums) {
            int tmp = curr;
            curr = Math.max(prev + n, curr);
            prev = tmp;
        }
        return curr;
    }

    @Test
    public void test5() {
        Assert.assertEquals(4, rob(new int[]{1, 2, 3, 1}));
        Assert.assertEquals(12, rob(new int[]{2, 7, 9, 3, 1}));
        Assert.assertEquals(2, rob(new int[]{1, 2}));
    }	
```

&emsp;&emsp;在分析出状态转移方程后，现在完整地概括下以上逐步优化代码的思路（套路）。

1. 设计暴力递归算法，找到冗余
2. 设计并存储状态（一维，二维，三维数组，甚至map）
3. 使用递归式（状态转移方程）
4. 自底向上计算最优解（其中可能涉及到空间复杂度的优化）

&emsp;&emsp;既然基本套路已经掌握，那么来练习下用动态规划解决斐波拉契数列问题。`F(0) = 0，F(1) = 1，F(n) = F(n-1) + F(n-2)（n>=2，n∈N*）`

```java
	public long fibonacci(int n) {
        if (n <= 0) {
            return 0L;
        }
        if (n == 1) {
            return 1L;
        }
        long pre1 = 0L;
        long pre2 = 1L;
        // 递推
        for (int i = 2; i <= n; i++) {
            long tmp = pre2;
            pre2 = pre1 + pre2;
            pre1 = tmp;
        }
        return pre2;
    }

    // 递归实现(涉及到太多重复计算,性能较差)
    public long fibonacci1(int n) {
        if (n <= 0) {
            return 0L;
        }
        if (n == 1) {
            return 1L;
        }
        return fibonacci1(n - 1) + fibonacci1(n - 2);
    }
```

&emsp;&emsp; 是不是很简单呢？摸索出原问题的解如何由子问题组合而成，就能把状态转移方程写出来，然后就套路一波，使用**自顶向下**的递归方法或者**自底向上**的递推方法，问题就迎刃而解啦。这里比较下这两种方式：

**自顶向下**

1. 能方便的得到递归公式，并用递归函数实现
2. 保持了递归实现的代码结构，逻辑上容易理解
3. 过程中只计算需要计算的子结果
4. 当采用了caching技术时多次被调用时天然的复用之前被调用时的子结果（比如连续两次计算斐波拉契数F(4)，F(5)，则计算F(5)时已知F(3)和F(4)，直接相加即可）

**自底向上**

1. 需要设计数据结构来完成自底向上的计算过程，逻辑上相对不那么直
2. 常常可以进行空间复杂度的优化（比如斐波拉契数列可以优化为只使用两个变量的额外存储空间，0-1背包问题可以优化为O(n)的空间复杂度）
3. 若进行了空间优化，则连续多次被调用时无法复用之前被调用的子结果

### leetcode 322（Coin Change）

&emsp;&emsp;再来一题目，热身一下。

&emsp;&emsp;`leetcode`第322题：给定不同面额的硬币 coins 和一个总金额 amount。编写一个函数来计算可以凑成总金额所需的最少的硬币个数。如果没有任何一种硬币组合能组成总金额，返回 -1。你可以认为每种硬币的数量是无限的。

&emsp;&emsp;示例一:

```
	输入: coins = [1, 2, 5], amount = 11
	输出: 3 
	解释: 11 = 1 + 5 + 5
```

&emsp;&emsp;示例二:

```
	输入: coins = [2], amount = 3
	输出: -1
```

&emsp;&emsp;乍一看，一脸懵逼，套用之前解决动态规划问题的思路，还是按步骤先定义状态，再写出状态转移方程。可以分析出，选取硬币时，要么选用当前idx位置的硬币，要么不选用当前idx位置的硬币。

- 状态：F(idx，amount)表示遍历到coins数组第idx个位置时且剩余总金额为s时，使用最少的硬币个数。
- 状态转移方程：F(idx, s) = min(F(idx,  s - coins[idx]) + 1, F(idx + 1, s)) (idx < coins.length && s > =0)

```java
	// 缓存计算结果数组
    private static int RESULT[][] = null;

    public int coinChange1(int[] coins, int amount) {
        RESULT = new int[coins.length][amount + 1];
        for (int i = 0; i < coins.length; i++) {
            for (int j = 0; j <= amount; j++) {
                RESULT[i][j] = Integer.MAX_VALUE;
            }
        }
        return search1(0, amount, coins);
    }

    public int search1(int idx, int s, int[] coins) {
        if (s == 0) {
            return 0;
        }
        if (idx >= coins.length || s < 0) {
            return -1;
        }
        if (RESULT[idx][s] != Integer.MAX_VALUE) {
            return RESULT[idx][s];
        }
        // 选取idx硬币
        int a = search1(idx, s - coins[idx], coins);
        a = a == -1 ? -1 : a + 1;
        // 不选idx硬币
        int b = search1(idx + 1, s, coins);
        RESULT[idx][s] = a != -1 && b != -1 ? Math.min(a, b) : Math.max(a, b);
        return RESULT[idx][s];
    }
```

&emsp;&emsp;写出上述代码后，继续思考如何用递推的方式优化空间复杂度，因为这种写法开辟了一个二维数组，空间复杂度是很高，虽然能够解决问题，那么到底有没有必要呢？

&emsp;&emsp;`leetcode`的解决方案给出的状态转移方程是：`F(s) = F(s−c) + 1`，其中s表示剩余总金额，c表示最后选取的硬币金额。根据这一状态转移方程，可以写出如下代码：

```java
	// 自顶向下
    public int coinChange(int[] coins, int amount) {        
        if (amount < 1) return 0;
        return coinChange(coins, amount, new int[amount]);
    }

    private int coinChange(int[] coins, int rem, int[] count) {
        if (rem < 0) return -1;
        if (rem == 0) return 0;
        if (count[rem - 1] != 0) return count[rem - 1];
        int min = Integer.MAX_VALUE;
        for (int coin : coins) {
            int res = coinChange(coins, rem - coin, count);
            if (res >= 0 && res < min)
                min = 1 + res;
        }
        count[rem - 1] = (min == Integer.MAX_VALUE) ? -1 : min;
        return count[rem - 1];
    }
```

```java
	// 自底向上
    public int coinChange(int[] coins, int amount) {
        int[] dp = new int[1 + amount];
        Arrays.fill(dp, -1);
        dp[0] = 0;
        for (int i = 1; i <= amount; ++i) {
            int min = Integer.MAX_VALUE;
            // find the minimum one.
            for (int c : coins) {
                if (i >= c && -1 != dp[i - c]) {
                    min = Math.min(min, dp[i - c]);
                }
            }
            if (Integer.MAX_VALUE != min) {
                dp[i] = 1 + min;
            }
        }

        return dp[amount];
    }
```

&emsp;&emsp;上述代码是不是和之前题目自顶向下以及自底向上的写法很像呢，区别也很明显，多了一级for循环遍历，所以这是一道搜索的动态规划问题。相比我之前的写法，二维数组降低为一维数组保存中间状态结果确实是很大的空间优化，注意这里自底向上的递推写法，根据状态转移方程，无法像之前题目一样继续优化空间复杂度。另外，可以看出**一个问题可能有多种不同的状态定义和状态转移方程定义**，选用合理的状态定义和状态转移方程可以避免无意义的空间消耗。

&emsp;&emsp;引用网上大神的语录作为结尾，**一个问题是该用递推、贪心、搜索还是动态规划，完全是由这个问题本身阶段间状态的转移方式决定的！**

- 每个阶段只有一个状态->递推
- 每个阶段的最优状态都是由上一个阶段的最优状态得到的->贪心
- 每个阶段的最优状态是由之前所有阶段的状态的组合得到的->搜索
- 每个阶段的最优状态可以从之前某个阶段的某个或某些状态直接得到而不管之前这个状态是如何得到的->动态规划
