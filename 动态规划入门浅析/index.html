content

动态规划（DP）

  什么是动态规划？动态规划的本质是对问题状态的定义和状态转移方程的定义。动态规划通过拆分问题，定义问题状态和状态之间的关系，使得问题能够以递推（或者说分治）的方式去解决。

  从上述描述可以看出，拆分问题才是动态规划的核心。而拆分问题，靠的就是状态的定义和状态转移方程的定义。

  写到这里，不仅身为读者的你很懵逼，我自己也挺懵逼的，但是毕竟写文章嘛，先搞点装逼的术语镇场面，下面从实际问题出发，再来感知这些概念。

  在分析出状态转移方程后，现在完整地概括下以上逐步优化代码的思路（套路）。

  既然基本套路已经掌握，那么来练习下用动态规划解决斐波拉契数列问题。F(0) = 0，F(1) = 1，F(n) = F(n-1) + F(n-2)（n>=2，n∈N*）

   是不是很简单呢？摸索出原问题的解如何由子问题组合而成，就能把状态转移方程写出来，然后就套路一波，使用自顶向下的递归方法或者自底向上的递推方法，问题就迎刃而解啦。这里比较下这两种方式：

自顶向下

  1. 能方便的得到递归公式，并用递归函数实现
  2. 保持了递归实现的代码结构，逻辑上容易理解
  3. 过程中只计算需要计算的子结果
  4. 当采用了caching技术时多次被调用时天然的复用之前被调用时的子结果（比如连续两次计算斐波拉契数F(4)，F(5)，则计算F(5)时已知F(3)和F(4)，直接相加即可）

自底向上

  1. 需要设计数据结构来完成自底向上的计算过程，逻辑上相对不那么直
  2. 常常可以进行空间复杂度的优化（比如斐波拉契数列可以优化为只使用两个变量的额外存储空间，0-1背包问题可以优化为O(n)的空间复杂度）
  3. 若进行了空间优化，则连续多次被调用时无法复用之前被调用的子结果

  乍一看，一脸懵逼，套用之前解决动态规划问题的思路，还是按步骤先定义状态，再写出状态转移方程。可以分析出，选取硬币时，要么选用当前idx位置的硬币，要么不选用当前idx位置的硬币。

  • 状态：F(idx，amount)表示遍历到coins数组第idx个位置时且剩余总金额为s时，使用最少的硬币个数。
  • 状态转移方程：F(idx, s) = min(F(idx, s - coins[idx]) + 1, F(idx + 1, s)) (idx < coins.length && s > =0)

  引用网上大神的语录作为结尾，一个问题是该用递推、贪心、搜索还是动态规划，完全是由这个问题本身阶段间状态的转移方式决定的！

  • 每个阶段只有一个状态->递推
  • 每个阶段的最优状态都是由上一个阶段的最优状态得到的->贪心
  • 每个阶段的最优状态是由之前所有阶段的状态的组合得到的->搜索
  • 每个阶段的最优状态可以从之前某个阶段的某个或某些状态直接得到而不管之前这个状态是如何得到的->动态规划
